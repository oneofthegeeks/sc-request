SC Request Flow — Step by Step

Both options cover the same steps — the difference is where it lives

Option A — Salesforce Native
Triggered from Opportunity Record
1
AE clicks "Request SC" button
Button on Opportunity record header or Sales Engineering Info section
2
Lightning modal opens
SC auto-selected based on AE→SC pairing. AE can override if needed.
3
Calendar availability shown
M365 API queried for AE + SC free/busy. Next 5 open slots shown, plus mini calendar grid.
4
AE selects request type & product line
Dropdowns: Demo / Discovery / etc. + GoTo Connect / Resolve / etc.
5
AE fills in request details
Freeform text: current state, what they're solving, demo requirements
6
AE submits
Single click. All actions fire automatically.
✓
Automated outcomes
• SC_Request__c record created
• Event created on Opportunity
• SC added to Opportunity Team
• Opp fields updated (SC name, date, checkbox, notes)
• Email sent to AE, SC, both managers
Option C — Slack Workflow
Triggered from Slack — /request-sc
1
AE types /request-sc in Slack
Works from any channel or DM. Slack modal opens instantly.
2
AE searches for Opportunity
Type-ahead search against Salesforce Opportunities by name or ID
3
SC auto-selected, AE can override
Pairing lookup same as Option A
4
Request type, product line, details
Dropdowns + text area in the Slack modal
5
Calendar availability
Bot posts available slots as buttons in a follow-up message. AE taps to confirm time.
6
AE submits
Confirmation message posted to AE's DM and optionally a team channel
✓
Automated outcomes
• Same SFDC writes as Option A
• Slack DM to SC with full details
• Email to AE, SC, both managers
• Confirmation message in Slack
